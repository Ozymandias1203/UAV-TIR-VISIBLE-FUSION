# matching_algorithm.md

## 1. 文档目的

本文档定义本项目跨光谱匹配层的设计。该层以 TWMM 为核心，用于在去畸变后的 RGB 与热红外图像之间建立稳定对应关系，并输出单应性矩阵与质量指标，为热信息投影到三维点云提供几何桥梁。

## 2. 设计原则

- 生产实现必须以 TWMM 为核心，不重新实现论文主算法。
- 优先使用 TWMM-main/TAMM_clean 作为生产入口。
- results_SIFT_SURF_RIFT_SCB_HOPC.py 仅作为比较与实验参考。
- 匹配层只负责跨光谱对应关系，不承担热辐射校正和三维重建。
- 输入必须是几何上已校正的图像，否则对应点和单应性会受到畸变干扰。

## 3. 输入与输出

### 3.1 输入

- 去畸变后的 RGB 图像
- 去畸变后的热红外图像
- 可选 ROI、初始尺度信息或裁剪信息
- 相关标定参数和图像尺寸信息

### 3.2 输出

- 对应点集合
- 内点 / 外点标记
- 单应性矩阵
- 匹配置信度或质量评分
- 诊断可视化图
- 运行时间与版本信息

## 4. TWMM 核心流程

TWMM 在本项目中保留以下四个核心环节：

### 4.1 原子块相似度图

- 将图像局部划分为原子块。
- 对原子块执行带权模板匹配，计算局部相似度图。
- 原子块应足够小，以降低局部形变对匹配的影响。

### 4.2 金字塔相似度图

- 使用多层局部最大池化构建更大尺度的相似度图。
- 大尺度图用于提供更强的鲁棒性，小尺度图用于提高定位精度。
- 通过多尺度组合平衡“稳”和“准”。

### 4.3 最大索引回溯

- 自顶向下回溯峰值位置，逐层细化对应点。
- 回溯过程应保留每一层的候选位置和峰值信息。
- 回溯结果构成最终的对应点候选集。

### 4.4 异常值剔除与单应性估计

- 对候选对应点进行几何一致性检查。
- 识别并剔除外点。
- 基于内点集合估计单应性矩阵。
- 输出必须同时包含内点统计和外点比例。

## 5. 适配器职责

TWMM 生产适配器应负责以下工作：

- 读取已经去畸变并预处理过的图像对。
- 将图像转换为 TWMM 所需的输入格式。
- 执行必要的尺度或裁剪对齐。
- 调用 TWMM 的生产入口。
- 将 TWMM 输出转换为统一的 `MatchResult` 语义对象。
- 将诊断信息写入日志和可视化文件。

适配器不得篡改 TWMM 的核心算法流程，只能做输入输出转换和参数组织。

## 6. 结果语义

### 6.1 对应点

对应点应至少包含：

- RGB 图像坐标
- 热红外图像坐标
- 是否为内点
- 所属金字塔层级
- 局部相似度或置信信息

### 6.2 单应性矩阵

- 单应性矩阵必须与对应点集合同时保存。
- 单应性矩阵应视作当前图像对的几何近似映射。
- 若后续发现匹配质量不足，应优先回到对应点质量层排查，而不是盲目修改投影层。

### 6.3 质量指标

建议至少输出：

- 对应点总数
- 内点数和外点数
- 外点比例
- 单应性稳定性指标
- 图像覆盖率
- 叠加可视化的人工检查结果

## 7. 预处理与前提

- 输入图像必须已经去畸变。
- 如需进一步增强对比度，应在匹配前的预处理层完成，而不是在 TWMM 内部临时拼接。
- 热红外图像与 RGB 图像的尺度差异应在匹配前尽量被规范化。
- 若图像对的重叠区域过小，应提前标记为低质量输入。

## 8. 失败模式

- 热红外图像纹理过弱。
- 图像对重叠不足。
- 目标场景存在过大视差或局部遮挡。
- 输入图像尚未去畸变。
- 图像尺度差异未被规范化。
- 过少的内点导致单应性不稳定。

遇到上述情况时，应优先输出低置信结果和诊断信息，而不是伪造高质量匹配。

## 9. 质量门槛

- 对应点数量应满足后续投影需求。
- 外点比例应处于可接受范围。
- 单应性矩阵在邻近图像对之间应保持相对稳定。
- 叠加可视化应能通过人工快速审查。

## 10. 依赖与边界

- 依赖于 calibration 层输出的去畸变图像和参数。
- 为 enrichment 层提供单应性矩阵和对应点质量指标。
- 不依赖 Metashape 项目状态。
- 不直接读取或修改温度矩阵。

## 11. 非目标

- 不是重新实现 TWMM 论文中的所有中间细节。
- 不是把 SIFT / SURF / RIFT 等方法当作生产主链路。
- 不是在匹配层直接做三维重建或温度赋值。
