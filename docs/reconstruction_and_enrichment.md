# reconstruction_and_enrichment.md

## 1. 文档目的

本文档定义 Metashape 重建、三维重投影、可见性判断、热信息富集以及结果导出的联合设计。该部分是本项目从二维双光谱数据走向三维温度点云的最后主链路。

## 2. 适用范围

- 适用于基于 RGB 影像的 SfM / MVS 重建。
- 适用于点云和相机位姿的几何重投影记录。
- 适用于把由 TIFF 热红外主输入帧生成的温度矩阵投影并融合到三维点上。
- 适用于导出带温度属性的点云和质检材料。
- 不负责辐射校正本身，也不负责 TWMM 的对应点生成。

## 3. 重建层设计

### 3.1 职责

Metashape 层只负责几何重建，不承担热辐射计算。其任务是把高分辨率 RGB 影像重建成可供后续富集的三维几何骨架。

### 3.2 典型流程

1. 通过 `Metashape.app.document` 获取或创建项目文档，并让项目文件与 `run_id` 绑定。
2. 创建或选择 chunk，导入 RGB 图像。
3. 挂接 calibration、dataset_profile 和 runtime_config 中定义的相机、路径与质量阈值引用。
4. 调用 `chunk.matchPhotos(...)` 建立初始匹配，再调用 `chunk.alignCameras()` 生成相机位姿。
5. 如果质量门槛通过，再调用 Metashape 的相机优化接口。
6. 调用 `chunk.buildDepthMaps(...)` 构建深度图。
7. 调用 `chunk.buildPointCloud(...)` 生成稠密点云；`buildDenseCloud()` 只作为历史名称参考，不作为 2.2.1 的规范名称。
8. 导出点云、相机、重投影记录和质检材料。

### 3.3 约束

- 该层应单独封装，不让通用算法层直接依赖 Metashape 细节。
- 该层的输入必须来自已经验证过的校准和数据管理流程。
- 该层只使用 RGB 影像作为几何主干，热红外 TIFF 影像不进入 SfM / MVS 主链路。
- 所有 Metashape 调用都必须集中在 metashape 适配层中，geometry 和 enrichment 只能消费导出的结果，不能直接访问 `Metashape.app`。
- 项目、chunk、相机、点云和导出摘要都必须和 `run_id` 一一对应。

## 4. 重投影与可见性设计

### 4.1 重投影记录

重投影记录描述三维点在 RGB 图像中的投影位置、误差和观测状态。它是可见性判断和热富集的前置几何证据。

### 4.2 需要保存的信息

- `point_id`
- `camera_id`
- `projected_x`
- `projected_y`
- `reprojection_error_px`
- `visibility_state`
- `occlusion_state`
- `source_chunk`

### 4.3 可见性状态

建议至少区分以下几类：

- 未观测
- 可见
- 遮挡
- 边界外
- 低置信

### 4.4 设计原则

- 可见性判断必须与重投影误差一起解释。
- 遮挡状态不能仅靠二维偏移判断，应结合三维几何和深度关系。
- 记录应保留所有支持视角，而不是只保留最终参与融合的视角。

## 5. 热信息富集设计

### 5.1 基本思路

对于每个三维点，先通过 RGB 几何重投影确定其在 RGB 图像中的位置，再利用匹配阶段得到的单应性矩阵把该位置映射到热红外图像坐标，最后从由 TIFF 热红外主输入帧生成的温度矩阵中采样温度并进行多视角融合。

### 5.2 采样策略

- 采用双线性插值处理热红外分辨率与几何点坐标之间的亚像素关系。
- 若点位于图像边界或无效区域，应标记为不可用，而不是外推出不可信温度。
- 如一个点在多个视角中都有可用温度，应进行融合而不是简单平均。

### 5.3 融合策略

建议融合时综合以下因素：

- 是否可见
- 重投影误差大小
- 点到光学中心的距离
- 对应匹配质量
- 是否靠近热图边缘或无效区域

融合权重不要求写死为单一公式，但必须可解释、可调参并可追溯。

### 5.4 结果字段

- `point_id`
- `xyz`
- `rgb`
- `temperature`
- `temperature_unit`
- `support_view_count`
- `fusion_weight`
- `temperature_quality_score`
- `source_thermal_frame_ids`

## 6. 导出设计

### 6.1 三维点云导出

- 优先输出可被常用点云工具读取的格式，例如 PLY 或 LAS。
- 须把温度作为显式属性保存。
- 须保留颜色、点编号和质量属性。

### 6.2 辅助导出

- 对应点叠加图
- 重投影误差直方图
- 温度覆盖率图
- 典型点的多视角采样明细
- 导出摘要 JSON

### 6.3 导出约束

- 导出的主结果和诊断材料应具有同一 run_id。
- 若某些点缺少温度，应显式标记缺失，而不是静默删除。
- 导出文件应尽量与 file_formats.md 的契约保持一致。

## 7. 质量门槛

- 点云密度应满足可视化和分析需求。
- 重投影误差应处于可接受范围。
- 温度覆盖率应足够高，且缺失区域分布合理。
- 融合结果应通过人工检查叠加图和统计图确认没有明显系统偏差。

## 8. 失败模式

- Metashape 重建失败或相机对齐质量不佳。
- 点云稀疏导致可用观测过少。
- 遮挡判断错误，导致错误温度被赋值到不可见点。
- 单应性矩阵质量差，导致热图采样位置偏移。
- 热红外 TIFF 图像边界或无效值过多，导致覆盖率过低。

## 9. 与其他层的接口

### 9.1 依赖输入

- reconstruction 层依赖 calibration、matching、radiometry、dataset_profile 和 runtime_config 的输出。
- geometry / visibility 依赖 Metashape 项目和点云结果。
- enrichment 依赖温度矩阵和单应性矩阵。

### 9.2 输出给下游

- 带温度属性的点云
- 几何和温度质量报告
- 可视化检查材料

## 10. 非目标

- 不是把 Metashape 的所有功能暴露为通用接口。
- 不是替代 Metashape 自己的重建算法。
- 不是在三维层面重新实现热辐射校正。
